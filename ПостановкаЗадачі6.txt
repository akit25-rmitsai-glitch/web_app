Постановка задачі

Необхідно розробити односторінковий веб-застосунок для резервування кімнат у готелі. Основною сторінкою застосунку буде index.php, а взаємодія між клієнтською та серверною частинами надалі повинна відбуватися з використанням AJAX.

Фронтенд частина застосунку має бути реалізована засобами HTML5, CSS3 та JavaScript. Для прискорення розробки інтерфейсу планувальника доцільно використати бібліотеку DayPilot Scheduler, а для спрощення роботи зі сценаріями та AJAX-запитами — бібліотеку jQuery.

Бекенд частина застосунку повинна бути реалізована мовою PHP. Для зберігання даних необхідно використати СУБД MySQL та організувати доступ до даних через PDO.

У системі потрібно передбачити:

відображення переліку кімнат готелю;
відображення стану кімнат;
фільтрацію кімнат за місткістю;
відображення статусів бронювання;
створення, редагування, видалення бронювань;
зміну тривалості бронювання та перетягування блоку бронювання;
контроль недопущення накладання бронювань.

На даному етапі розробки необхідно створити структуру бази даних, таблиці rooms і reservations, наповнити їх тестовими даними та підготувати базовий файл головної сторінки застосунку.

Висновок

У ході виконання лабораторної роботи було здійснено постановку задачі для розробки веб-застосунку резервування кімнат у готелі. Було визначено основні функціональні можливості майбутньої системи, а також розподілено ролі між використаними технологіями: HTML5 і CSS3 для створення інтерфейсу, JavaScript і jQuery для клієнтської логіки та AJAX-взаємодії, PHP для серверної частини, MySQL для зберігання даних.

Також було створено базу даних, дві основні таблиці rooms і reservations, та виконано їх наповнення тестовими записами. Додатково підготовлено головний файл застосунку з базовою структурою сторінки та стилями, що створює основу для подальшої реалізації функціоналу бронювання.

Отриманий результат є достатнім для переходу до наступних етапів розробки, проте на поточному етапі застосунок ще не реалізує повний функціонал взаємодії з базою даних та календарним планувальником, що буде предметом наступних лабораторних робіт.
